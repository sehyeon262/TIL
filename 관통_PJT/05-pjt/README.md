### 💻 개인별 관심 종목 관리 (05-PJT)
본 프로젝트는 '관심 종목 토론 데이터 수집' 프로젝트를 확장하여, 사용자별 회원 기능과 관심 종목 관리 기능을 추가하는 것을 목표로 합니다. Django의 인증 시스템을 기반으로 사용자를 관리하고, 사용자별 데이터를 저장 및 처리하는 웹 서비스의 핵심 구조를 학습합니다.


# ✨ 주요 기능
### 🧑‍💻 회원 관리 기능

Django의 기본 인증 시스템(django.contrib.auth)을 활용한 회원가입, 로그인, 로그아웃 기능을 구현했습니다. 
로그인 상태에 따라 네비게이션 바 메뉴가 동적으로 변경됩니다. 
@login_required 데코레이터를 사용하여 로그인한 사용자만 접근할 수 있는 페이지를 보호합니다.


### ⭐ 관심 종목 관리 (CRUD)

사용자별로 관심 있는 주식 종목을 등록하고 관리할 수 있습니다.
추가(Create): 프로필 페이지에서 새로운 관심 종목을 폼을 통해 추가할 수 있습니다. 
조회(Read): 사용자가 등록한 모든 관심 종목을 프로필 페이지에서 목록 형태로 조회할 수 있습니다. 
삭제(Delete): 각 관심 종목 옆의 삭제 버튼을 통해 DB에서 해당 데이터를 삭제하고 화면을 갱신합니다. 
관심 종목이 없을 경우, "등록된 관심 종목이 없습니다." 라는 메시지를 표시합니다. 


### 📈 기존 크롤링 기능 연동
프로필 페이지의 관심 종목을 클릭하면, 이전 프로젝트에서 구현한 크롤링 기능이 실행됩니다. 
요청한 종목의 댓글 데이터가 DB에 이미 존재하면 DB 데이터를 우선 사용하고, 없을 경우에만 새로 크롤링을 수행하여 서버 부하를 최소화합니다. 


### 🎨 UI/UX 개선
Bootstrap을 적용하여 전체 페이지의 디자인 통일성을 높이고 사용자 친화적인 UI를 구성했습니다. 
회원가입, 로그인, 프로필 페이지의 폼과 목록을 부트스트랩 컴포넌트를 활용해 깔끔하게 스타일링했습니다.


### 🛠️ 기술 스택
Backend: Python, Django
Frontend: HTML, CSS, Bootstrap5
Crawling: Selenium
Database: SQLite3



### 📖 학습 내용 및 느낀 점
1. Django 인증 시스템에 대한 이해
    Django가 제공하는 강력한 인증 시스템을 직접 사용해보며 웹 서비스의 기본적이면서도 핵심적인 회원 관리 로직을 이해할 수 있었습니다. @login_required 데코레이터를 통해 특정 페이지 접근을 제어하는 방식이 매우 편리하다고 느꼈습니다.

2. 데이터 무결성
    Meta 클래스의 UniqueConstraint를 이용해 한 명의 사용자가 동일한 종목을 중복으로 저장할 수 없도록 제약 조건을 설정하며 데이터베이스의 무결성을 지키는 방법을 학습했습니다.

3. URL Namespace의 중요성
    프로젝트를 진행하며 NoReverseMatch 오류를 여러 번 마주했습니다. 
    이 경험을 통해 urls.py에 app_name을 지정하여 URL 네임스페이스를 설정하는 이유와 {% url '앱이름:URL이름' %} 형태로 명확하게 경로를 지정하는 방식의 중요성을 깨달았습니다. 
    앱 단위로 URL을 관리하면 프로젝트 규모가 커져도 이름 충돌 없이 안정적으로 경로를 관리할 수 있다는 것을 배웠습니다.

4. 부트스트랩을 활용한 프론트엔드 스타일링
    단순한 HTML 태그만으로는 사용자가 보기 좋은 화면을 만들기 어렵다는 것을 느끼고 부트스트랩을 적극적으로 활용했습니다. 
    특히 input-group을 이용해 폼 요소를 보기 좋게 배치하는 등 미리 만들어진 컴포넌트를 활용하여 빠르고 효율적으로 UI를 개선하는 방법을 익혔습니다.
