# 01-pjt



## 학습한 내용
이번 프로젝트를 통해 외부 API(OpenWeatherMap, 금융상품통합비교공시 API) 를 활용하여 데이터를 수집하고, 이를 분석 및 가공하는 일련의 과정을 학습했습니다. 단순히 API를 연동하는 데 그치지 않고, 데이터의 구조를 이해하고 필요한 정보를 추출해내며, 목적에 맞게 데이터를 재가공하는 방법에 대해 집중적으로 익혔습니다.

또한 실제 서비스를 구현한다는 관점에서, 수집한 데이터를 바탕으로 사용자가 체감할 수 있는 맞춤형 날씨 서비스나 금융 상품 비교 기능을 기획하고 설계하는 경험을 했습니다.

### F01. Key 값 출력
내용: OpenWeatherMap API에서 받아온 JSON 데이터를 파싱하고, 그 구조를 이해한 후 key 값들을 출력함으로써 전체 데이터의 흐름을 파악함.

학습: JSON 구조 탐색 방법, dict.keys() 활용, 중첩된 JSON 내부 탐색

### F02. 원하는 값 추출
내용: 날씨 정보 중에서 main, weather 등 특정 key에 해당하는 값만 추출

학습: 중첩된 dictionary 내에서 원하는 key 접근, 리스트 안의 딕셔너리 처리 방법 (weather[0]['main'] 등)

### F03. Key 한글화
내용: 추출한 key 값을 이해하기 쉬운 한글로 변환하여 새롭게 구성된 dictionary 생성

학습: key_map을 활용한 key 이름 치환 방법, dictionary comprehension 사용법

### F04. 섭씨 온도 추가
내용: F03에서 만든 dictionary에 온도 관련 데이터를 섭씨 단위로 변환하여 추가

학습: 켈빈(K) → 섭씨(℃) 변환 공식 적용 (temp - 273.15), 새로운 필드 추가 방법

### F05. 생성형 AI 활용
내용: OpenWeatherMap API를 활용한 프로젝트를 구성하고, 이를 설명하는 프롬프트 작성

학습: API 활용 목적에 맞는 질문 설계 및 사용자 중심 프롬프트 작성법 경험

## 어려웠던 부분
이번 프로젝트에서 가장 어려웠던 부분은 API 응답 데이터의 key 값을 한글로 바꾸는 F03: Key 한글화 기능 구현이었습니다.

처음에는 if문과 for문을 사용하여 각 key마다 조건을 주고 하나씩 변경하는 방식으로 접근했습니다. 하지만 이 방식은 코드가 길고 반복적인 구조를 가졌으며, 다양한 형태의 중첩 구조(JSON 안에 딕셔너리와 리스트가 섞인 경우)를 처리하기에는 한계가 있었습니다. 유지보수나 확장성 측면에서도 비효율적이라는 문제를 느꼈습니다.

딕셔너리 내부 key를 key_map을 기준으로 한글로 변환하며, 재귀함수를 사용하여 JSON 구조 안에 있는 중첩된 딕셔너리와 리스트까지도 처리할 수 있도록 설계했습니다. map() 함수와 딕셔너리의 특징, 재귀 구조에 대한 이해가 필요했기 때문에 학습량이 많았고 구현에 시간도 많이 소요되었습니다.

그러나 이 과정을 통해 단순한 기능 구현을 넘어 복잡한 데이터 구조를 효율적으로 처리하는 방법과 더 나은 코드 구조를 위한 사고방식을 배울 수 있었던 점에서 큰 의미가 있었습니다.

## 새로 배운 것들
1. 딕셔너리 변환 및 Key 한글화 처리 방법
- key를 한글로 바꾸기 위한 key_map 구조 설계
- dict comprehension과 map() 함수 활용
- 재귀 함수를 사용하여 중첩된 데이터까지 한 번에 변환하는 방법 학습

2. 코드 효율성과 가독성 개선
- 반복적인 조건문 대신 구조적이고 재사용 가능한 함수형 설계
- 한 번의 재귀 함수로 다양한 깊이의 데이터를 처리할 수 있도록 리팩토링
- 실무에 가까운 수준의 코드 작성 방식 이해


## 느낀점
이번 프로젝트를 통해 단순한 기능 구현을 넘어서 외부 API 연동, 데이터 분석 및 가공, 효율적인 코드 작성까지 다양한 실전 경험을 할 수 있었습니다. 특히 OpenWeatherMap과 같은 실제 API를 다루면서, 데이터 구조를 이해하고 필요한 정보만을 추출해 가공하는 과정이 얼마나 중요한지 체감하게 되었습니다.

처음에는 JSON 형태의 복잡한 데이터 구조를 해석하고, 원하는 값을 추출하거나 변환하는 데 어려움이 많았지만, 반복적인 실습과 코드를 통한 경험으로 점차 구조화된 사고를 익힐 수 있었습니다. 특히 Key 한글화 기능에서는 단순 반복문이 아닌 딕셔너리와 재귀 함수, 그리고 map 함수 활용을 통해 더 효율적이고 확장성 있는 코드를 설계하는 중요성을 배웠습니다.

또한, 단순히 정답을 도출하는 것이 아니라 사용자 입장에서 데이터를 어떻게 보여줄지, 어떻게 의미 있게 가공할지 고민하게 되면서 서비스 설계적 사고를 키우는 계기가 되었습니다.

하지만 아직 부족한 점이 많고, 스스로 완벽하게 코드를 작성해내기 위해서는 많은 노력과 함수 공부 그리고 알고리즘 공부 등이 필요하다는 것을 뼈저리게 느꼈습니다.
꾸준히 스터디도 하고 시간을 많이 들여 마스터하겠다고 다짐했습니다!
